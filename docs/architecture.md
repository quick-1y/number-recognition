# Архитектура системы автоматического распознавания автомобильных номеров

Документ фиксирует целевую архитектуру решения согласно ТЗ.

## Модули обработки видеопотока

1. **Ingest** – подключение к RTSP/ONVIF и виде файлам, управление переподключением.
2. **VideoDecoder** – аппаратное (NVDEC/VAAPI) либо программное декодирование.
3. **Motion/Trigger** – определение движения/триггеров.
4. **Detector** – YOLOv8/YOLOv11 для детекции госномеров.
5. **Tracker** – ByteTrack/SORT, расчёт направления движения.
6. **Cropper** – выделение региона госномера.
7. **OCR Engine** – модуль распознавания символов (EasyOCR, PaddleOCR, CRNN).
8. **Postprocessor** – голосование по символам, нормализация, антидубликаты.
9. **Rules Engine** – применение списков, расписаний, сценариев.
10. **Event Manager** – запись событий в PostgreSQL и MinIO, генерация webhooks.
11. **Webhook Service** – гарантированная доставка уведомлений.
12. **Alarm Relay Controller** – управление тревожными выходами камер.
13. **Admin/API/UI** – интерфейсы администратора и оператора.
14. **Monitoring/Logging** – Prometheus, Grafana, JSON-логи.

## Хранилища

- **PostgreSQL** – конфигурация, события, аудит.
- **MinIO (S3)** – изображения и клипы.
- **Prometheus/Grafana** – метрики и наблюдаемость.

## Схема API

REST API разделён по доменам: каналы, списки, события, сценарии, webhooks, диагностика. Авторизация – OAuth2/JWT, роли admin/operator/viewer, маскирование номеров для viewer.

## Мониторинг

Метрики Prometheus: задержка, FPS, ошибки OCR, dropped frames. Health-checkи `/api/diagnostics/{health,ready,live}`.

## Развёртывание

- Docker Compose – для локальных окружений.
- Kubernetes – целевое окружение (описание Helm chart в будущих итерациях).
- GPU – через `nvidia-container-runtime`.

## Расширяемость

Проект подготовлен для развития: подключение ReID, PTZ, расширенные шаблоны номеров, кастомные OCR движки и т.д.
